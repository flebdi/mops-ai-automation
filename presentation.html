The Problem

Today, everything is done by hand

A person reads every campaign request and manually copies the information into Asana, Salesforce, and Pardot. This is slow, error-prone, and invisible.

⏱️

Days of delay

Each request sits in a queue waiting for a human to process it before anything can move forward.

✏️

Naming errors

Campaign names get formatted inconsistently, which breaks reporting and Pardot–Salesforce sync.

🔗

Systems don't talk

Asana, Salesforce, and Pardot are disconnected. A human is the only bridge between them.

👁️

No visibility

There's no easy way to see where a request is stuck or whether the sync between systems is healthy.

The Solution

One form submission triggers
the whole pipeline

The team fills out a single Asana form. The automation reads it, validates it, and creates everything that needs to exist — in every system — automatically.

Step by Step

What happens after
a form is submitted

A1
Automatic

Read & Understand the Request

The system reads the Asana form, figures out the campaign type (Event, Webinar, Email…), the region, the go-live date, and who should own it. If anything is missing or unclear, it flags it for a human instead of guessing.

A5
Gate — must pass before next step

Check the Campaign Name

The campaign name must follow a strict format: 2026_EMEA_Webinar_DrupalSecurity_Q3. If it doesn't match, the system suggests a corrected name and waits for a human to approve it before touching Salesforce. Nothing gets created with a bad name.

A2
Automatic

Build the Salesforce Campaign

Once the name is approved, the system creates the Salesforce campaign with all the right fields, sets up the member statuses (Registered, Attended, No Show…), links any parent campaign, and connects it to Pardot — all in one go.

A3
Automatic

Create the Asset Checklist

Based on the campaign type, the system creates every subtask the team needs in Asana — landing page, email invites, follow-up cadences, UTM parameters — so nothing gets forgotten.

A4
Automatic

Write the Campaign Brief

The system drafts a one-page campaign brief (objective, audience, messaging, KPIs, timeline) and attaches it to the Asana task and the Salesforce campaign record automatically.

A6
Daily watchdog — runs on a schedule

Monitor Sync Health

Every day, the system checks that Pardot and Salesforce member counts match. If they've been out of sync for more than 2 hours, it sends a Slack alert to the right regional owner. Every Monday it sends a full health report.

Visibility

Three ways to track
what's happening

Every campaign request is traceable from submission to completion — no black boxes.

📋

Asana

The original form task is updated with subtasks and comments. The regional owner is assigned automatically.

🖥️

Trigger.dev Dashboard

Every pipeline run is visible here. The team can see which step completed, which failed, and re-run it — no code required.

📊

Airtable Audit Log

Every AI decision is recorded here — what the system decided, how confident it was, and whether it routed to a human.

💬

Slack Alerts

The right regional owner gets a Slack message when something needs their attention — a sync issue, a low-confidence triage, or an approval request.
